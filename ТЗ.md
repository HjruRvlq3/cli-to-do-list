## Функциональные требования

### 1. Работа с пунктами (Task Management)

- Создание новой задачи
- Отметка выполнения задачи
- Отмена отметки выполнения задачи
- Редактирование текста задачи
- Удаление задачи
- Удаление всех выполненных задач
- Сохранение задач между запусками

### 2. Выводы (Output)

- Вывод всех задач
- Вывод невыполненных задач
- Вывод выполненных задач
- Поддержка команд:
  - `help` / `h` — вывод справки по командам
  - `exit` — завершение работы программы

### 3. Ввод (Input)

- Обработка команд пользователя
- Ввод текста задачи
- Подтверждение при удалении всех выполненных задач или одной не выполненной задачи

## Взаимодействие пакетов

1. **main.go** — точка входа.
2. **app** — содержит функцию `Run()`, которая управляет запуском программы и координирует взаимодействие input/service/output.
3. **input.go** — получает и валидирует команды пользователя.
4. **service** — обрабатывает бизнес-логику, вызывает функции создания, редактирования, отметки и удаления задач.
5. **task.go** — определяет структуру задачи и конструкторы/методы для работы с отдельными задачами.
6. **taskBase.go** — отвечает за сохранение и загрузку задач в JSON, обеспечивает их сохранность между запусками.
7. **output.go** — отвечает за вывод информации и сообщений пользователю.

## Требования к пользовательскому опыту

- Простое добавление, редактирование и удаление задач
- Понятный вывод статуса задач `[✓]` для выполненных, `[ ]` для невыполненных
- Подтверждение при удалении всех выполненных задач или одной не выполненной задачи
